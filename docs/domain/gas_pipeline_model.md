# Модель тестового объекта: участок газопровода

## Назначение

Этот документ описывает демонстрационный объект для MVP системы подбора МТР/КСМ. Объект нужен, чтобы тестировать систему не на разрозненных строках, а на понятном инженерном сценарии: есть участок газопровода, у него есть условия эксплуатации, детали, паспорта, Excel-каталог и правила подбора аналогов.

Система носит рекомендательный характер. Она не принимает окончательное инженерное решение вместо эксперта, а сокращает область поиска до наиболее похожих кандидатов и показывает, на какие параметры нужно обратить внимание.

## Тестовый объект

Объект: участок технологического газопровода.

Условная среда эксплуатации:

- рабочая среда: природный газ;
- возможные примеси: H2S и CO2;
- режим: эксплуатация в условиях низких температур;
- требование: материалы и покрытия должны быть проверены на совместимость со средой и условиями эксплуатации.

Для MVP можно использовать два варианта среды:

1. Обычный газ без подтвержденной агрессивной среды.
2. Газ с H2S/CO2, где отсутствие подтверждения по среде должно давать статус "требует проверки".

## Детали объекта

| Деталь | Пример | Зачем нужна в MVP |
| --- | --- | --- |
| Труба | Труба DN159, стенка 10, 09Г2С | Базовый элемент газопровода |
| Отвод 90 градусов | ОКШ90 DN159, стенка 10, К48 | Проверка угла, DN, толщины, материала |
| Отвод 45 градусов | ОКШ45 DN159, стенка 10, К48 | Проверка отличия угла и составных замен |
| Отвод 30 градусов | ОГ30 DN325, стенка 8, К48 | Проверка составной замены 3x30 = 90 |
| Задвижка | Задвижка DN150 PN40 | Другой класс оборудования, не отвод |
| Заглушка | Заглушка DN25 PN160 | Проверка типа изделия и давления |
| Переход | Переход DN219/DN159 | Проверка двух диаметров |
| Тройник | Тройник DN159/DN89 | Проверка сложной геометрии |

## Важные параметры

| Группа | Параметры | Почему важны |
| --- | --- | --- |
| Идентификация | Код МТР, код КСМ, наименование, условное обозначение | Нужны для поиска в корпоративной номенклатуре |
| Тип изделия | труба, отвод, задвижка, заглушка, переход, тройник | Совсем другой тип обычно нельзя считать прямым аналогом |
| Геометрия | DN, D1, D2, толщина стенки, угол | Ошибка в геометрии может сделать изделие неприменимым |
| Давление | PN/Ру, рабочее давление, испытательное давление | Давление ниже требуемого является серьезным риском |
| Материал | марка стали, класс прочности | Влияет на прочность и совместимость со средой |
| Среда | газ, H2S, CO2, агрессивная среда | Для H2S/CO2 отсутствие подтверждения должно уходить эксперту |
| Покрытие | внутреннее, наружное, тип покрытия | Не отклонять автоматически, но обязательно подсвечивать |
| Нормативка | ГОСТ, ТУ, ЛНД | Нужна для правил и комментариев эксперту |
| Источники | паспорт, страница, Excel-строка, ЛНД | Каждый вывод должен быть проверяемым |

## Рекомендательная логика

В MVP система должна не столько "запрещать", сколько помогать эксперту быстрее сузить поиск.

Пример поведения:

1. Пользователь ищет отвод 90 градусов DN159 К48 для H2S.
2. Система извлекает карточку запроса.
3. Система ищет похожие МТР/КСМ в каталоге.
4. Система показывает Top-20 кандидатов.
5. Если у кандидата не подтверждено покрытие или H2S, система не удаляет его, а добавляет предупреждение.
6. Эксперт принимает финальное решение.

## Минимальные демо-сценарии

### Сценарий 1. Поиск по запросу

Запрос: "Нужен отвод 90 DN159 стенка 10 К48 для H2S".

Ожидаемое поведение:

- система формирует карточку запроса;
- находит отводы DN159;
- отвод 90 градусов должен быть выше отвода 45 градусов;
- кандидаты без подтверждения H2S получают предупреждение.

### Сценарий 2. Проверка аналога

Заявлено: ОКШ90 DN159 стенка 10 К48.

Кандидат: ОКШ45 DN159 стенка 10 К48.

Ожидаемое поведение:

- система оставляет кандидата в выдаче только если режим поиска допускает широкий список;
- добавляет предупреждение: "угол отличается";
- статус: "требует проверки" или "низкая релевантность".

### Сценарий 3. Покрытие не подтверждено

Заявлено: требуется внутреннее покрытие.

Кандидат: покрытие не указано.

Ожидаемое поведение:

- не удалять кандидата автоматически;
- добавить предупреждение;
- добавить комментарий эксперту: "Проверить наличие внутреннего покрытия по паспорту или ЛНД".

### Сценарий 4. Составная замена

Запрос: 1 отвод 90 градусов.

Кандидат: 2 отвода 45 градусов или 3 отвода 30 градусов.

Ожидаемое поведение:

- показать как составную замену;
- не считать прямым аналогом;
- отправить на экспертную проверку.
