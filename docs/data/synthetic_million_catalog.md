# Синтетический каталог на 1 000 000 изделий

## Распределение

| Коллекция | Количество | Доля |
|---|---:|---:|
| трубы | 260 000 | 26% |
| отводы | 300 000 | 30% |
| переходы | 160 000 | 16% |
| арматура | 180 000 | 18% |
| заглушки | 100 000 | 10% |

Распределение хранится в
`data/generation/million_items_distribution.json`. Это начальная модель
нагрузки, а не статистика реального предприятия. После получения выгрузки МТР
доли нужно пересчитать по фактическим данным.

## Почему файл не хранится в Git

Миллион JSON-карточек занимает много места и создаёт лишние конфликты в ветках.
В Git хранится генератор, конфигурация и тесты. Полный набор воспроизводится с
одинаковым `seed`, поэтому у обоих разработчиков будут одинаковые данные.

Команда для полного набора:

```powershell
python -m backend.app.scripts.generate_synthetic_catalog
```

Быстрая проверка на 10 000 карточках:

```powershell
python -m backend.app.scripts.generate_synthetic_catalog --count 10000 --output data/generated/mtr_catalog_10k.jsonl
```

## Гарантии генератора

- точное число карточек и заданное распределение классов;
- уникальные `card_id`, коды МТР и КСМ;
- структура ItemCardV2 с динамическими `properties`;
- разные характеристики для разных классов изделий;
- раздельные значения `true`, `false` и `null`;
- потоковая запись: миллион карточек не загружается в память одновременно;
- одинаковый результат при одинаковом `seed`.

Технически одинаковые характеристики у двух позиций допустимы: это могут быть
разные учётные позиции, партии или КСМ. Полными дубликатами считаются записи с
одинаковым идентификатором или кодом; генератор такие записи не создаёт.
